## E2E #8 v12 fixture — Notes app (starting state)

This repo is the **starting state** for the E2E #8 v12 run: the agents
grow it into a full-stack **Notes** app from the spec in the platform
repo (`tools/staging-smoke-fixtures/notes-fullstack-spec.md`). Do not
pre-build the API/UI here — the point of the E2E is the agents building
it. What's planted for the run:

- **`lodash@4.17.20`** (CVE-2021-23337) — genuinely imported by the slug
  helper in `src/lib/strings.ts`, so the security-driven bump to
  `>= 4.17.21` is unavoidable. `npm audit fix --force` wrongly suggests a
  non-existent `4.18.x`; the correct, agent-verified bump is `4.17.21`.
- **vitest** (`npm test`) with a slug-helper unit test — `npm test` exits
  0 on a clean checkout, and the test pins the helper so the lodash bump
  is verified to keep `kebabCase` working.
- **Playwright** (`npm run test:e2e`) with a `webServer` config that
  builds + serves the app, plus a harness smoke test (`e2e/smoke.spec.ts`).
  The agents add the real Notes UI e2e tests; QA drives this same running
  app for its browser acceptance checks.
- The base scaffold builds clean with `DATABASE_URL` unset (the Notes
  store is in-memory — no DB at runtime), so it's deployable to Vercel
  for the demo.

## Pre-installed

- **Next.js 15** App Router + TypeScript + Tailwind + Turbopack
- **Prisma 6** + Postgres datasource, with NextAuth's four required
  tables (User/Account/Session/VerificationToken) seeded in
  `prisma/schema.prisma`
- **NextAuth v5 (beta)** with the **Resend** email provider + Prisma
  adapter. Auth helpers exposed via `src/lib/auth.ts` →
  `auth() / signIn() / signOut() / handlers`
- The `[...nextauth]` API route wired to those handlers

## To run locally

1. Copy `.env.example` → `.env.local` and fill in `DATABASE_URL`,
   `AUTH_SECRET` (`openssl rand -base64 32`), `AUTH_RESEND_KEY`,
   `RESEND_FROM`.
2. `npm install`
3. `npx prisma migrate dev --name init` to create the auth tables.
4. `npm run dev`

## What the coder agent does on a generated repo

- Extends `prisma/schema.prisma` with project-specific models below
  the NextAuth tables (those four are required by the Prisma adapter
  and must not be removed).
- Builds pages/components under `src/app/...`.
- Adds Vercel env vars via the platform UI (Apex's `provision_vercel_project`
  links the Vercel project to this repo automatically).
